# 🍞 BreadItNow 프로젝트 - 포트폴리오 자기소개서

## 📌 프로젝트 개요

### 프로젝트명
**BreadItNow (빵잇나우)**

### 프로젝트 설명
빵 알림, 예약 및 관리 서비스 - 베이커리의 신선한 빵 출시 알림, 고객 예약, 배송 관리를 통합으로 제공하는 플랫폼

### 프로젝트 기간
2024 ~ 2025년 (진행 중)

---

## 🎯 내 역할 및 기여도

### 담당 영역
이 프로젝트에 참여하면서 다음 영역들을 경험했습니다:

#### 1️⃣ **마이크로서비스 아키텍처 설계**
- Spring Cloud 기반 분산 시스템 구축
- API Gateway를 통한 통합 라우팅 관리
- 8개의 독립적인 마이크로서비스 구성
  - 인증 서비스 (Auth API)
  - 고객 서비스 (Customer API)
  - 사업자 서비스 (Owner API)
  - 예약 서비스 (Reservation API)
  - 알림 서비스 (Notification API)
  - Redis 캐시 레이어
  - 공통 라이브러리 (Common)

#### 2️⃣ **백엔드 개발 (Java/Spring Boot)**

**기술 스택:**
- Java 기반 개발
- Spring Boot 3.4.2
- Spring Cloud Gateway 4.2.0
- Spring Data Redis
- Gradle 빌드 자동화

**구현한 주요 기능:**

**인증 서비스 (breaditnow-auth-api)**
- JWT 토큰 기반 사용자 인증
- 고객/사업자 역할 구분 및 권한 관리
- Redis를 활용한 세션 관리
- 보안 토큰 검증 및 리프레시 로직

**고객 서비스 (breaditnow-customer-api)**
- 베이커리 정보 조회 및 필터링
- 신규 빵 출시 알림 구독/취소
- 추천 상품 시스템
- 마이페이지 (프로필, 주소, 알림 설정)
- 즐겨찾기 관리

**사업자 서비스 (breaditnow-owner-api)**
- 베이커리 정보 등록/수정/삭제
- 상품 관리 (등록, 수정, 삭제, 이미지 업로드)
- 가격 및 재고 관리
- 배송 가능 지역 설정
- 매출 통계 및 분석 대시보드

**예약 서비스 (breaditnow-reservation-api)**
- 상품 예약 신청 로직
- 예약 상태 관리 (5단계 워크플로우)
- 예약 취소 및 변경 처리
- 배송 상태 추적
- 실시간 예약 알림

**알림 서비스 (breaditnow-notification-api)**
- Firebase Cloud Messaging (FCM) 통합
- 푸시 알림 발송
- 알림 수신 동의 관리
- 알림 유형별 커스터마이징

#### 3️⃣ **프론트엔드 개발 (TypeScript/React)**

**기술 스택:**
- TypeScript
- React 19.0.0
- Next.js 15.1.6 (서버사이드 렌더링)
- Tailwind CSS (스타일링)
- Zustand (상태관리)
- React Query (서버 상태 관리)
- Framer Motion (애니메이션)
- Next PWA (모바일 앱 경험)

**개발한 주요 페이지:**

**고객 앱 (BreadItNow_FE)**
- **홈 화면**: 추천 베이커리, 인기 상품, 검색/필터링
- **베이커리 상세**: 정보, 실시간 재고, 리뷰, 영업시간
- **상품 상세**: 갤러리, 설명, 가격, 리뷰, 예약 버튼
- **예약 프로세스**: 
  - 단계별 예약 (상품선택 → 날짜시간 → 배송방식 → 결제)
  - 달력 UI를 이용한 직관적인 날짜 선택
  - Daum 주소 API 통합 (배송주소 입력)
- **마이페이지**: 프로필, 예약이력, 즐겨찾기, 알림설정
- **알림센터**: 예약/재입고/이벤트 알림 관리

#### 4️⃣ **인프라 및 시스템 설계**

**아키텍처 특징:**
- 마이크로서비스 아키텍처 (MSA)
- API Gateway 패턴
- Redis 캐싱 레이어
- 비동기 메시징 (예약 알림)
- 실시간 상태 추적

**핵심 워크플로우:**

**고객의 예약 프로세스:**
```
앱 접속 → 베이커리 검색 → 상품 선택 → 예약 신청 
→ 배송 방식 선택 → 결제 → 예약 완료 
→ 실시간 상태 추적 → 픽업/배송 완료
```

**사업자의 상품 관리:**
```
로그인 → 상품 정보 입력 → 이미지 업로드 
→ 가격/재고 설정 → 출시 시간 설정 
→ 상품 등록 → 고객 알림 발송
```
