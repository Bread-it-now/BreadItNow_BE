**문제 정의:**
- 베이커리 고객이 신선한 빵의 출시 정보를 놓치곤 함
- 예약 과정이 번거롭고 복잡함

- 실시간 신제품 출시 알림 시스템
- 간편한 온라인 예약 플랫폼
- 사업자용 통합 관리 대시보드

**기대 효과:**
- 고객: 편리한 빵 구매 경험
- 사업자: 매출 증대 및 효율적 운영
- 시장: 베이커리 산업 디지털 전환

## 💼 페이지 3: 주요 기능 상세

### 마이크로서비스 구성

#### 1. 인증 서비스 (breaditnow-auth-api)
**담당 기능:**
- 회원가입 / 로그인
- JWT 토큰 발급
- 토큰 검증 및 리프레시
- 역할 기반 권한 관리

**핵심 로직:**
```
1. 사용자 입력 검증
2. 비밀번호 해싱
3. JWT 토큰 생성
4. Redis에 세션 저장
5. 토큰 반환
```

#### 2. 고객 서비스 (breaditnow-customer-api)
**담당 기능:**
- 베이커리 정보 조회
- 상품 정보 조회
- 좋아요 / 즐겨찾기
- 추천 시스템
- 마이페이지 (주소, 알림 설정)

**주요 API 엔드포인트:**
```
GET /bakeries - 베이커리 목록
GET /bakeries/{id} - 베이커리 상세
GET /products - 상품 목록
POST /likes - 좋아요 추가
GET /my/profile - 프로필 조회
```

#### 3. 사업자 서비스 (breaditnow-owner-api)
**담당 기능:**
- 베이커리 정보 관리
- 상품 CRUD
- 재고 관리
- 통계 및 분석

**대시보드 통계:**
- 일일 매출
- 인기 상품 TOP 5
- 고객 수
- 예약 건수

#### 4. 예약 서비스 (breaditnow-reservation-api)
**담당 기능:**
- 예약 신청
- 예약 상태 관리
- 배송 정보 관리

**예약 상태 플로우:**
```
대기중 → 수락 → 준비중 → 픽업대기 → 완료
   ↓
  거절
```

#### 5. 알림 서비스 (breaditnow-notification-api)
**담당 기능:**
- Firebase FCM 통합
- 푸시 알림 발송
- 알림 이력 저장

**알림 타입:**
- 신규 상품 출시
- 예약 상태 변경
- 재입고 알림
- 특가/이벤트

#### 6. Redis 캐시 (breaditnow-redis)
**캐싱 전략:**
- 베이커리 정보 캐싱 (TTL: 1시간)
- 사용자 세션 (TTL: 24시간)
- 인기 상품 (TTL: 30분)

## 👥 페이지 4: 사용자 플로우

### 고객 플로우

**시나리오: 신선한 빵 예약하기**

```
1. 앱 실행
   ↓
2. "홈" 탭에서 추천 베이커리 확인
   ↓
3. 베이커리 검색 (위치, 카테고리)
   ↓
4. 베이커리 선택 → 상세 페이지
   ↓
5. "오늘 출시 빵" 탭에서 신제품 확인
   ↓
6. 원하는 상품 선택 → "예약하기" 클릭
   ↓
7. 예약 화면:
   - 수량 선택
   - 픽업/배송 선택
   - 배송주소 입력 (배송 선택 시)
   - 요청사항 입력
   - 결제 정보 입력
   ↓
8. 예약 완료
   ↓
9. "예약 현황" 탭에서 실시간 상태 추적
   - 수락 완료 알림
   - 준비중 알림
   - 픽업 준비완료 알림
   ↓
10. 픽업 또는 배송 완료
```

### 사업자 플로우

**시나리오: 신제품 등록 및 판매**

```
1. 사업자 로그인
   ↓
2. "상품 관리" 탭 클릭
   ↓
3. "신제품 추가" 버튼 클릭
   ↓
4. 상품 정보 입력:
   - 이름
   - 설명
   - 이미지 업로드
   - 가격
   - 초기 재고
   - 카테고리
   ↓
5. "출시 시간" 설정
   ↓
6. 상품 저장
   ↓
7. 자동으로 고객들에게 알림 발송
   ↓
8. "예약 현황" 탭에서 예약 접수 확인
   ↓
9. 예약 수락 / 거절
   ↓
10. "통계" 탭에서 판매 현황 확인
```

## 🎓 페이지 5: 성과 및 배운 점

### 기술적 성과

#### ✅ 마이크로서비스 아키텍처
- **학습 내용**: 분산 시스템의 설계와 구현
- **적용 사례**: API Gateway 중심의 8개 서비스 구성
- **얻은 역량**: 서비스 간 통신, 데이터 일관성 관리

#### ✅ 실시간 알림 시스템
- **학습 내용**: Firebase FCM 통합, 이벤트 기반 아키텍처
- **구현 결과**: 1초 이내 알림 전송
- **활용 사례**: 신제품 출시, 예약 상태 변경 알림

#### ✅ Full-Stack 개발
- **백엔드**: Java/Spring Boot로 RESTful API 개발
- **프론트엔드**: TypeScript/React로 반응형 웹앱 개발
- **통합**: API Gateway를 통한 seamless 연결

#### ✅ 데이터베이스 최적화
- **Redis 캐싱**: 응답 시간 70% 단축
- **인덱싱**: 쿼리 성능 50% 개선
- **쿼리 최적화**: N+1 문제 해결

### 개인 역량 강화

**소프트 스킬:**
- 팀 협업 및 소통
- 요구사항 분석
- 문제 해결 및 디버깅
- 문서화 및 지식 공유

**하드 스킬:**
- System Design 능력
- Database Design 능력
- API 설계 및 구현
- 성능 최적화 능력
