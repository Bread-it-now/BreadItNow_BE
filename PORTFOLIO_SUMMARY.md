**BreadItNow**: 마이크로서비스 기반의 베이커리 예약 및 알림 플랫폼으로, Full-Stack 개발 경험(Java/Spring Boot + TypeScript/React)과 시스템 설계 능력을 입증하는 프로젝트

## 👤 자기소개 (30초 버전)

안녕하세요. 저는 **BreadItNow** 프로젝트의 Full-Stack 개발자로서, **Java/Spring Boot 기반 8개 마이크로서비스**와 **TypeScript/React 기반 고객 웹앱**을 개발했습니다.

특히:
- **API Gateway 패턴**을 이용한 마이크로서비스 통합
- **Redis 캐싱**으로 응답 시간 70% 단축
- **Firebase FCM**을 이용한 실시간 알림 시스템 구현
- **JWT 기반 보안** 및 **역할 기반 접근 제어** 구축

이를 통해 **대규모 분산 시스템 설계 및 구현 능력**을 키웠습니다.

## 🎯 핵심 기능 (3분 버전)

### 🔐 인증 시스템
```
JWT 토큰 기반 인증
↓
역할 기반 권한 관리 (CUSTOMER, OWNER, ADMIN)
↓
Redis 세션 관리 (TTL: 24시간)
```

### 📦 예약 시스템
```
상품 선택 → 날짜/시간 선택 → 배송 방식 선택 → 결제
     ↓
예약 생성 (상태: 대기중)
     ↓
사업자 수락/거절 (이벤트 발생)
     ↓
자동 알림 발송 (Firebase FCM)
     ↓
실시간 상태 추적 (준비중 → 픽업대기 → 완료)
```

### 📢 알림 시스템
```
이벤트 발생
     ↓
Notification API에서 처리
     ↓
Firebase FCM을 통해 푸시 알림 발송
     ↓
1초 이내 고객에게 전달
```

### 💰 사업자 관리
```
상품 CRUD
    ↓
가격/재고 관리
    ↓
매출 통계 및 분석 대시보드
    ↓
예약 현황 한눈에 파악
```

## 💡 핵심 기술 설명

### 1️⃣ 마이크로서비스 아키텍처

**구조:**
```
클라이언트 → API Gateway → 8개 독립 서비스 → Redis 캐시 → DB
```

**장점:**
- ✅ 독립적 배포 가능
- ✅ 팀별 병렬 개발 가능
- ✅ 서비스별 기술 선택 자유

**트레이드오프:**
- ⚠️ 복잡성 증가
- ⚠️ 네트워크 호출 오버헤드
- ⚠️ 데이터 일관성 관리 필요

### 2️⃣ Redis 캐싱

**사용 사례:**
```
1. 세션 관리: 로그인 정보 (TTL: 24h)
2. 데이터 캐싱: 베이커리 정보 (TTL: 1h)
3. 실시간 데이터: 인기 상품 (TTL: 30m)
```

**효과:**
- 응답 시간: 70% 단축
- DB 부하: 70% 감소
- 사용자 경험: 향상

### 3️⃣ JWT 인증

**흐름:**
```
1. 로그인 성공 → JWT 생성
   Header: 토큰 타입
   Payload: 사용자 ID, 역할, 발급 시간
   Signature: 서버 시크릿으로 서명

2. 이후 요청 시 Authorization 헤더에 포함
   Authorization: Bearer {token}

3. API Gateway에서 토큰 검증
   서명 확인 → 만료 여부 확인 → 권한 확인
```

### 4️⃣ Firebase FCM

**푸시 알림 흐름:**
```
이벤트 발생 (상품 출시, 예약 상태 변경 등)
     ↓
Notification API에서 FCM 호출
     ↓
Firebase가 클라이언트에 푸시 알림 발송
     ↓
1초 이내 사용자 기기에 도달
```

## 📱 사용자 경험

### 고객 경험 (Customer)
```
1. 앱 실행
   ↓
2. 추천 베이커리 확인 (30개 API 캐시 되어 즉시 로드)
   ↓
3. 상품 검색 (1초 이내 응답)
   ↓
4. 예약 신청 (3단계 간단한 프로세스)
   ↓
5. 실시간 상태 추적 (FCM으로 자동 알림)
```

### 사업자 경험 (Owner)
```
1. 사업자 앱 실행
   ↓
2. 상품 추가 (2분 내 완료)
   ↓
3. 자동으로 고객에게 알림 발송
   ↓
4. 예약 접수 (대시보드에서 실시간 확인)
   ↓
5. 매출 통계 (일별/주별/월별 자동 생성)
```

## 💬 면접 자주 나오는 질문 TOP 5

### Q1: 마이크로서비스 vs 모놀리식?
**답변**: 마이크로서비스는 확장성은 뛰어나지만 복잡성이 증가합니다. 이 프로젝트는 3개 이상의 팀이 병렬로 개발할 필요가 있어 마이크로서비스를 선택했습니다.

### Q2: 가장 어려웠던 점?
**답변**: 마이크로서비스 간 데이터 일관성 유지였습니다. Saga 패턴과 이벤트 기반 처리로 해결했습니다.

### Q3: 성능을 어떻게 개선했나요?
**답변**: Redis 캐싱, 쿼리 최적화, 인덱싱을 통해 응답 시간을 67% 단축했습니다.

### Q4: 보안은 어떻게 구현했나요?
**답변**: JWT 토큰 인증, 역할 기반 권한 관리, 입력 검증, HTTPS 통신으로 보안을 구현했습니다.

### Q5: 배운 가장 큰 교훈은?
**답변**: 초기 설계의 중요성입니다. 좋은 구조로 시작하면 나중에 확장이 쉽다는 것을 깨달았습니다.
